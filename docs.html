Quickstart

Set the base URL to CC Agent and call it with your virtual key. Anthropic SDKs and any client that reads ANTHROPIC_BASE_URL pick it up with no other change.

export ANTHROPIC_BASE_URL=https://api.ccagent.example

curl "$ANTHROPIC_BASE_URL/v1/messages" \
  -H "Authorization: Bearer cca_live_..." \
  -H "content-type: application/json" \
  -d '{
    "model": "claude-sonnet",
    "max_tokens": 1024,
    "messages": [
      { "role": "user", "content": "Hello from CC Agent" }
    ]
  }'

API compatibility

Two surfaces, one gateway. The Anthropic Messages API is live today. The OpenAI surface is served by a translation layer that is still in progress, so test it before you cut over.

  • Anthropic Messages, POST /v1/messages (available)
  • OpenAI Chat Completions, POST /v1/chat/completions (translation layer, in progress)

Routing modes

A declarative policy decides the upstream per request. Pick the mode per project and the gateway handles failover for you.

  • Balanced, spread traffic across healthy regions for throughput
  • Pinned, lock a project to one region for residency
  • Failover-first, prefer a primary and fall back automatically

Models and regions

The live catalog is Anthropic Claude on Amazon Bedrock, with Anthropic Direct for failover. Scope each project to the models it is allowed to call.

  • Claude Opus, Sonnet, and Haiku on Amazon Bedrock
  • Regions: us-east-1, ap-northeast-1, eu-central-1
  • Set a model allowlist per project, for example claude-*

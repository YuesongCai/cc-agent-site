How it's built

Built to be reconciled.

Two systems do the work, like a bank: one carries every model request fast, the other holds the rules and the records. Nothing about your spend is invented after the fact — every charge traces back to the model's own count.

Data plane

The fast lane

The part your agents actually talk to. It carries each request to the model and streams the answer straight back, untouched, while quietly applying your limits.

Go · stateless
Control plane

The rule book

A separate system that decides who may call which model, sets the budgets, vets each customer, and runs the billing — kept away from live traffic.

NestJS control plane
State

The record room

Where settings, usage history, running cost totals, and credentials are kept — detailed enough to reconcile every charge, with secrets held in a managed vault.

PostgreSQL · ClickHouse · Redis · secrets vault
Egress

The model shelf

The connections out to the models you're cleared to use, built to add new providers without a rebuild. Claude on Amazon Bedrock and Anthropic Direct are live today.

Pluggable adapters · open vendor schema
  1. 01

    Check the caller

    Each call arrives with a key tied to a real, vetted company — and we only ever store a scrambled copy of that key, never the original.

  2. 02

    Hold the line

    Before anything runs, we check the call against its rate limit and its monthly budget, so spend can't quietly run away.

  3. 03

    Send it through

    The call goes to the model you chose for that job, and the reply streams back to your agent exactly as the model sent it.

  4. 04

    Count it honestly

    We record the real usage straight from the model's own response and tag it, so every line on your bill can be traced back and checked.

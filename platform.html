How it's built

Built to be reconciled.

Two systems do the work, like a bank: one carries every model request fast, the other holds the rules and the records. Every charge traces back to the model's own count.

Tap any node for the technical detail →

Your agents
CC Agent Gateway
Data planeThe fast lane

The part your agents actually talk to. A stateless Go proxy that carries each request to the model and streams the answer back untouched, while applying rate limits and budget checks in-line.

  • Stateless Go — built for throughput
  • Streams responses untouched from the model
  • Rate limits and budget checks in the data path
Control planeThe rule book

A separate system that decides who may call which model, sets the budgets, vets each customer, and runs the billing — kept away from live traffic.

  • NestJS control plane, isolated from traffic
  • Manages keys, budgets, customer vetting
  • Runs billing and reconciliation
StateThe record room

Where settings, usage history, running cost totals, and credentials are kept — detailed enough to reconcile every charge, with secrets held in a managed vault.

  • PostgreSQL + ClickHouse for relational and analytics
  • Redis for rate-limit counters
  • Managed secrets vault for credentials
EgressThe model shelf

The connections out to the models you're cleared to use, built to add new providers without a rebuild. Claude on Amazon Bedrock and Anthropic Direct are live today.

  • Pluggable adapters with open vendor schema
  • Claude on Bedrock + Anthropic Direct live
  • New providers without a rebuild
